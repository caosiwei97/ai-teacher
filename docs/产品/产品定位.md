# AI Teacher — 产品定位

> 本文档是 AI Teacher 项目的核心上下文。AI 在参与本项目任何阶段的工作时，都应先阅读本文档以理解项目全貌。

---

## 项目一句话描述

构建一个 **AI 1v1 私教平台**：用户上传学习资料（或使用 AI 自有知识），系统自动生成知识图谱和学习路径，通过苏格拉底式追问引导用户真正掌握知识。

---

## 产品灵感与参考

### 核心参考：同类竞品

通过实际交互分析，同类苏格拉底教学产品的关键产品设计：

| 模块 | 设计细节 |
|------|----------|
| **诊断摸底** | 3 道选择题快速判断用户水平，决定起点 |
| **知识图谱 Roadmap** | 线性节点序列（5-15 个），每节点有标题+描述，可视化进度 |
| **苏格拉底追问** | 禁止直接给答案，顺着用户回答追问 2-3 轮暴露盲区，总结后放行 |
| **掌握度门控** | 每个节点需 80% 掌握度才解锁下一节点，无法蒙混过关 |
| **评估卡片** | 节点完成后渲染总结+回顾表格+核心标签 |
| **教学模式切换** | 严格教练 / 宽松模式 |
| **快问功能** | 选中内容直接提问，结合上下文回答 |
| **AI 建议回复** | 帮助不知道怎么答的用户 |
| **会话管理** | 学习中 / 新对话 / 已完成三态 |

### 差异化：用户上传资料

类似 Google NotebookLM，用户可以上传自己的学习资料（PDF、Markdown、网页链接），AI 基于这些资料构建教学内容和追问。没有上传资料时，使用 AI 自有的世界知识。

---

## 目标用户

- 有明确学习目标的成年人（不限于技术领域）
- 希望深度理解而非被动阅读的学习者
- 需要个性化学习路径的人

## 核心场景

1. **用户上传学习资料** → 系统解析资料 → 生成知识图谱
2. **诊断摸底** → 判断用户水平 → 确定学习起点
3. **苏格拉底式教学** → 追问引导 → 暴露盲区 → 真正掌握
4. **掌握度评估** → 节点通过 → 解锁下一节点
5. **学习回顾** → 跨会话记忆 → 个性化调整

---

## 核心概念

### 学习会话 (Session)

一次完整的学习交互，包含多个节点的教学对话。

### 知识图谱 (Roadmap)

根据学习资料或主题自动生成的学习路径，每个节点是一个可评估的知识点。

### 掌握度 (Mastery)

每个知识点的掌握程度（0-100%），≥80% 标记为已掌握。

### 学习资料 (Source)

用户上传的 PDF / Markdown / 网页链接，作为教学内容的基础。

### 苏格拉底追问 (Socratic Tutoring)

不直接给答案，通过追问引导用户自己发现理解中的漏洞。

---

## 系统需要具备的能力

1. **学习资料管理**：上传、解析、向量化存储
2. **知识图谱生成**：根据资料/主题自动生成学习路径
3. **诊断评估**：快速判断用户当前水平
4. **苏格拉底式教学**：追问引导，不直接给答案
5. **掌握度评分**：每轮对话后评估理解程度
6. **会话记忆**：跨会话记住用户画像、易错点、学习偏好
7. **可视化进度**：知识图谱节点状态、进度条

## 约束与边界

### MVP 阶段（v0.1）

- 单用户本地使用（不需要多租户）
- 不需要支付/订阅系统
- 不需要移动端
- Docker 一键启动

### 技术约束

- 用户在本地验证测试
- 中间件通过 Docker 镜像拉取，不手动安装
- 所有服务通过 docker-compose 一键启动

---

## 非目标（MVP 不做）

- 多用户/多租户
- 支付/订阅
- 移动端适配
- 语言学习（听说）
- 多模态（视频/音频）
- 课程管理后台
