# AI Teacher — 产品定位

> 本文档是 AI Teacher 项目的核心上下文。AI 在参与本项目任何阶段的工作时，都应先阅读本文档以理解项目全貌。

---

## 项目一句话描述

构建一个 **AI 1v1 私教平台**：用户上传学习资料（或使用 AI 自有知识），系统自动生成知识图谱和学习路径，通过苏格拉底式追问引导用户真正掌握知识。

---

## 设计理念

1. **主动学习优于被动阅读** — 通过追问引导用户思考，而不是单向灌输知识
2. **基于用户资料教学** — 用户上传自己的 PDF/Markdown/网页链接，AI 基于这些资料构建教学内容；无资料时使用 AI 自有知识
3. **可量化的掌握度** — 每个知识点都有掌握度评分（≥80% 达标），确保真正理解后才继续
4. **结构化学习路径** — 自动将复杂主题分解为知识图谱，让学习者清楚知道学什么、学到哪

---

## 目标用户

- 有明确学习目标的成年人（不限于技术领域）
- 希望深度理解而非被动阅读的学习者
- 需要个性化学习路径的人

## 核心场景

1. **用户上传学习资料** → 系统解析资料 → 生成知识图谱
2. **诊断摸底** → 判断用户水平 → 确定学习起点
3. **苏格拉底式教学** → 追问引导 → 暴露盲区 → 真正掌握
4. **掌握度评估** → 节点通过 → 解锁下一节点
5. **学习回顾** → 跨会话记忆 → 个性化调整

---

## 核心概念

| 概念 | 说明 |
|------|------|
| **学习会话 (Session)** | 一次完整的学习交互，包含多个节点的教学对话 |
| **知识图谱 (Roadmap)** | 根据学习资料或主题自动生成的学习路径，每个节点是一个可评估的知识点 |
| **掌握度 (Mastery)** | 每个知识点的掌握程度（0-100%），≥80% 标记为已掌握 |
| **学习资料 (Source)** | 用户上传的 PDF / Markdown / 网页链接，作为教学内容的基础 |
| **苏格拉底追问** | 不直接给答案，通过追问引导用户自己发现理解中的漏洞 |

---

## 约束与边界

### MVP 阶段

- 单用户本地使用（不需要多租户）
- 不需要支付/订阅系统
- 不需要移动端
- Docker 一键启动

### 非目标（不做）

- 多用户/多租户
- 支付/订阅
- 移动端适配
- 语言学习（听说）
- 多模态（视频/音频）
- 课程管理后台
