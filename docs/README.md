## 文档结构

```
docs/
├── 产品/                          ← 产品定义（WHAT）
│   ├── 产品定位.md                 ← 项目一句话、核心场景、用户、约束
│   └── 需求规格.md                 ← 用户故事、功能规格、MVP 范围
├── 设计/                          ← 技术方案（HOW）
│   ├── 技术架构.md                 ← 技术栈、系统架构、数据模型
│   ├── API接口.md                  ← 已实现的接口文档
│   ├── Prompt设计.md               ← 苏格拉底 Prompt + Agent 工具定义
│   ├── Agent引擎优化方案.md        ← Agent 引擎重构方案（参考 4 个开源仓库）
│   └── 决策记录.md                 ← 关键技术决策及原因（ADR）
├── 开发/                          ← 迭代进展（WHEN & PROGRESS）
│   ├── 迭代计划.md                 ← 14 个迭代 + 分类标签 + 状态
│   ├── 开发日志.md                 ← 日志索引 + 里程碑 + 继续点
│   └── 日志/                       ← 按日期归档的详细开发日志
│       └── 2026-05-08/             ← 项目初始化 ~ 迭代 010
└── 测试/                          ← 测试相关
    └── e2e/                       ← E2E 测试用例
```
