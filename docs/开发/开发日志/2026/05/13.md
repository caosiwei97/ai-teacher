#### 14:00 — 沙箱错误传递 + 编辑器集成终端

**状态**：✅ 已完成

**事件**：
- Bug 修复：沙箱执行报错时 server 直接抛 500，前端只显示"执行失败: 500"无法定位原因。添加 try-catch 将错误详情（message）以 `{stderr, status: "Sandbox Error"}` 形式返回，前端可显示具体错误
- UI 改造：CodePanel 从条件显示输出改为 VSCode 风格集成终端 — 编辑器和终端上下分割，可拖拽调整大小（`react-resizable-panels`），终端始终可见
- 新增 `terminal-panel.tsx`：暗色终端面板，stdout 绿色/stderr 红色/沙箱错误 amber 警告，空闲时显示占位符，执行中显示 spinner

**决策**：沙箱错误走 HTTP 200 而非 500 — 因为前端 `useCodeExec` 对非 200 只显示 `执行失败: {status}`，丢失了错误详情。用 200 + `status: "Sandbox Error"` 让终端面板可以区分展示

#### 14:20 — 编辑器终端一体化 + execd 错误解析修复

**状态**：✅ 已完成

**事件**：
- UI 改进：去掉 react-resizable-panels 拖拽分割，改为编辑器和终端共享同一暗色背景、细线分隔的一体化面板（Jupyter cell 风格）
- Bug 修复（根因）：`parseExecdSse` 只解析 `type: "stdout"` 和 `type: "stderr"`，但 OpenSandbox execd 的运行时错误返回 `type: "error"` + `error.traceback[]`，完全被忽略。所有 Python 异常（ImportError、SyntaxError 等）都显示为"无输出"。增加 `type: "error"` 解析，将 traceback 清除 ANSI 转义码后写入 stderr

**决策**：不在沙箱镜像里预装 openai — 用户代码如果 import 失败，现在能正确看到 `ModuleNotFoundError` 而不是空白
