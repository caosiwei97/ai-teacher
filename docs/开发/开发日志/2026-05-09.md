## 21:00 — 迭代 017：架构拆分 Phase 2 — Worker 激活 + BullMQ 队列

**状态**：✅ 已完成

**事件**：
- 激活 Worker 为独立 BullMQ 消费进程（port :38423）
- 实现 chat-turn 队列：Hono POST 入队 → Worker 消费 → Agent 执行 → Redis Pub/Sub 推送 → Hono SSE 转发
- Prisma Message 新增 `status` 字段（sending → processing → completed / failed）
- 前端 chat hook 从 AI SDK `useChat` 重写为自定义 POST + SSE 双请求实现
- 删除 5 个已废弃的 standalone tool 文件 + stale dist/
- 新增 `pnpm dev:worker` 脚本，`pnpm dev` 现在启动三进程并行

**技术决策**：
- BullMQ + Redis Pub/Sub：解耦 API 层和 Agent 层，Worker 可独立扩展
- Worker 崩溃恢复：启动时扫描 processing 状态的消息，标记为 failed
- SSE 事件格式：text-delta / tool-call / tool-result / done / error（JSON over SSE）
- quick-question 和 suggest-reply 暂不走队列（非核心路径）

**代码变更**：
- `packages/db/prisma/schema.prisma` — Message 新增 status 字段（default "completed"）
- `packages/shared/src/schemas/message.ts` — 新增 MessageStatus enum
- `apps/worker/src/index.ts` — 重写为 BullMQ 启动入口 + 崩溃恢复 + 健康检查 HTTP
- `apps/worker/src/processors/chat-turn.ts` — 新增，核心队列消费者
- `apps/worker/src/lib/learner-profile.ts` — 新增，从 server 迁移的 buildLearnerProfile
- `apps/server/src/routes/chat.ts` — 重写为 POST 入队 + GET SSE 双端点
- `apps/server/src/services/queue.ts` — 新增，BullMQ Queue 实例
