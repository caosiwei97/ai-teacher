## 22:00 — 开发体验优化：pnpm 强制 + 一键启动 + 环境自动化

**状态**：✅ 已完成

**事件**：
- 用户反馈项目启动步骤繁琐，需要简化新人上手流程
- pnpm 版本管理从 Volta 管理到项目级约束
- 发现 `pnpm setup` 和 `pnpm init` 会与 pnpm 内置命令冲突，最终选用 `bootstrap`
- Docker 镜像拉取遇到国内网络问题，尝试 registry-mirrors + Docker Desktop proxy 均未生效
- 最终通过 Docker Desktop GUI 配置 `host.docker.internal:7891` 代理解决
- 发现 Next.js 运行时无法读取根目录 .env，需 symlink（`apps/web/.env → ../../.env`）
- 将 .env 复制 + symlink 创建自动化为 `scripts/setup-env.mjs`

**技术决策**：
- `npx only-allow pnpm` + `engines.pnpm: ">=10"` 强制使用 pnpm v10+
- `npm-run-all2`：`run-s` 串行、`run-p` 并行编排脚本
- `pnpm bootstrap`：`setup:env` → `docker:up` → `db:generate` → `db:migrate` → `db:seed`
- `pnpm dev`：`run-p dev:*` 并行启动 web + worker
- `scripts/setup-env.mjs`：自动从 .env.example 创建 .env + 创建子项目 symlink

**代码变更**：
- `package.json` — 新增 `preinstall`/`engines`/`bootstrap`/`dev`/`docker:up`/`docker:down`/`setup:env`/`db:seed` 脚本
- `package.json` — 新增 `npm-run-all2` devDependency
- `scripts/setup-env.mjs` — 新增环境自动化脚本
- `README.md` — 快速开始简化为 2 步 + 新增常用命令表 + Docker 镜像加速说明

**验证结果**：
- `pnpm bootstrap` 完整流程通过（docker up → db generate → migrate → seed）✅
- `node scripts/setup-env.mjs` 正确创建 .env 和 symlink ✅
- `pnpm dev` 并行启动 web + worker 成功 ✅
