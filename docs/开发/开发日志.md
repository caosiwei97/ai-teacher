#### 2026-05-09 23:30 — 迭代 023：技术栈全面升级（AI SDK v6 + Zod 4）

**状态**：✅ 已完成

**事件**：
- 5 个子任务并行迁移：packages/shared、packages/agent、apps/worker、apps/server、apps/web
- AI SDK codemod 超时，改为手动逐包迁移
- 依赖升级：`ai` 4.3→6.0、`@ai-sdk/openai` 1.3→3.0、`zod` 3.24→4.4、`@hono/zod-validator` 0.5→0.8、新增 `@ai-sdk/react` 3.0

**代码变更**：

AI SDK v4→v6 核心变更：
- `CoreMessage` → `ModelMessage`（5 文件）
- `CoreTool` → `Tool`（2 文件）
- `parameters` → `inputSchema`（所有 tool 定义）
- `maxSteps` → `stopWhen` + `stepCountIs()`（streamText 调用）
- 流事件 `textDelta`→`text`、`args`→`input`、`result`→`output`（tutor-graph.ts）
- `UIMessage` 移除 `content`/`annotations`，改用 `parts`/`metadata`（web 前端 3 文件）
- 新增 `getTextFromParts()` 辅助函数

Zod 3→4 核心变更：
- 所有 `import { z } from 'zod/v3'` → `'zod'`（14 文件）
- `z.record(z.unknown())` → `z.record(z.string(), z.unknown())`（message.ts）
- `err.flatten()` → `flattenError(err)`（error-handler.ts、sessions.ts）

其他：
- `packages/agent/src/tool-registry.ts`：`ToolDefinition.parameters` → `inputSchema`、`z.ZodSchema` → `z.ZodType`
- `apps/worker/src/agent/base-agent.ts`：新增 `LanguageModel` 显式返回类型
- `apps/web/src/hooks/use-chat-stream.ts`：新增 `MessageMetadata`、`AnnotationData` 类型导出
- `apps/web/src/components/chat/chat-area.tsx`：`msg.content` → `getTextFromParts(msg.parts)`、`msg.annotations` → `msg.metadata?.annotations`
- 6 个子包 `package.json` 版本号更新

**验证结果**：
- `pnpm install` 无 peer dependency 冲突 ✅
- `pnpm build`（全量）web + worker + server 构建成功 ✅
- E2E 测试：等 PostgreSQL 恢复后运行（Docker 容器需重启）

**下一步**：重启 Docker Desktop 后运行 `npx playwright test` 确认 E2E 全量通过

#### 2026-05-10 21:00 — Bug 修复：Diagnostic API 500 + LLM 切换 DeepSeek

**状态**：✅ 已完成

**事件**：
- 用户报告 `POST /api/sessions/:id/diagnostic` 返回 500
- 排查发现两个叠加问题：
  1. 智谱 GLM 通过 OpenAI 兼容接口返回的 JSON 被 markdown code fence 包裹，AI SDK `generateObject` 无法解析
  2. AI SDK `@ai-sdk/openai` provider 默认发送 `response_format: { type: 'json_schema' }`，智谱和 DeepSeek 都不支持此 OpenAI 专有格式
- 用户决定从智谱切换到 DeepSeek（DeepSeek-V4-Flash 模型）
- 发现 AI SDK 有专用 `@ai-sdk/deepseek` provider，原生支持 `generateObject`（使用 `response_format: { type: 'json_object' }`）
- 切换后 diagnostic 接口正常返回 200

**代码变更**：
- `.env` — LLM 配置从智谱切换到 DeepSeek（API key + base URL）
- `apps/worker/src/agent/provider.ts` — `createOpenAI` → `createDeepSeek`，移除 `stripCodeFences` 函数
- `apps/worker/src/agent/base-agent.ts` — 默认模型从 `glm-5-turbo` → `deepseek-v4-flash`
- 全局替换 8 处硬编码 `"glm-5-turbo"` → `"deepseek-v4-flash"`（worker 5 处 + server 2 处 + base-agent 1 处）
- 移除 `experimental_repairText: stripCodeFences`（3 个文件 5 处 `generateObject` 调用）
- 新增 `@ai-sdk/deepseek` 依赖到 worker 和 server 包
