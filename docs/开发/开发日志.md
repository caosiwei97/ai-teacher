  - **根因**：`provider-registry.ts` 的 `default` 分支用 `createOpenAI()`，而 `@ai-sdk/openai` v3 的 `provider(modelId)` 默认走 **Responses API**（`/responses` 端点）；智谱/通义/Kimi/MiniMax/小米等国产 OpenAI 兼容供应商只实现 `/chat/completions`（GitHub issue zai-org/GLM-5#39 证实智谱不支持 /responses），故上游返回"模型不存在/not found"。DeepSeek provider 默认走 chat 故不受影响。
