# 迭代 038 — 沙箱多文件工程结构

> 状态：✅ 已完成 | 优先级：P2 | 依赖：021, 037

## 背景

当前沙箱是单文件执行模式（OpenSandbox execd `/code` 端点），无法展示工程结构、多文件编辑。编程教学场景中，学生需要理解项目文件组织、模块化开发等概念。

## 目标

支持沙箱内的多文件工程结构，前端可查看和编辑多个文件。

## 方案

采用方案 A：OpenSandbox 文件管理 API（execd 文件读写接口 + PTY 终端 WebSocket 代理）。

### 架构

```
前端 (SandboxProvider + useSandboxApi)
  ↓ REST API
Hono Server (sandbox route 代理)
  ↓ HTTP Proxy
OpenSandbox execd (文件系统 + PTY)
```

## 实现细节

### 服务端
- `apps/server/src/routes/sandbox.ts` — Hono 代理路由（文件 CRUD + PTY + 执行）
- `apps/server/src/services/sandbox.ts` — 沙箱生命周期管理（创建/复用/清理）
- `safeJson()` helper 处理 execd 空 body 响应

### 前端
- `apps/web/src/contexts/sandbox-context.tsx` — SandboxProvider（文件树、打开文件、PTY 会话）
- `apps/web/src/hooks/use-sandbox-api.ts` — API 层（multipart metadata + file 格式）
- `apps/web/src/components/editor/sandbox-file-tree.tsx` — 文件树组件
- `apps/web/src/components/editor/sandbox-terminal.tsx` — xterm.js 交互终端
- `apps/web/src/components/sidebar/code-panel.tsx` — IDE 布局（Activity Bar + 文件树 + 编辑器 + 终端）

### Bug 修复（2026-05-15）
- 文件上传 multipart 格式：execd 需要 `metadata`(JSON) + `file` 两个 part
- 代理响应解析：execd 空 body 返回 `\n`，需 `.trim()` 后判空
