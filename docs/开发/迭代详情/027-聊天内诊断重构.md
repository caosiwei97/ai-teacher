# 027 — 聊天内诊断重构

> 状态：✅ 已完成 | 分类：🔵 功能 | 优先级：P0 | 依赖：026
> 完成日期：2026-05-10

## 目标

将诊断摸底从独立 REST API（页面外完成）改为聊天内 `askQuestion` 工具驱动的选择题交互。用户在聊天流中直接作答，提交后 Agent 自动生成路线图并开始教学。

## 背景

当前诊断流程使用独立的 `DiagnosticAgent` + `DiagnosticQuiz` 组件，需要 2 步 REST API 调用，在聊天页面外完成。这导致流程割裂、体验不连贯。改为聊天内交互后，诊断 → 路线图 → 教学形成无缝流。

## Checklist

### Phase 1：Shared Schema + 类型基础

- [x] 新增 `AskQuestionParams` Schema 到 `packages/shared/src/schemas/diagnostic.ts`
  ```typescript
  AskQuestionParams: {
    questions: Array<{
      id: string;           // e.g. "d1"
      question: string;
      title: string;        // Tab 标题，如 "核心定义"、"背景调查"
      options: Array<{ id: string; text: string }>;
    }>;
    nodeId: string;          // "diagnosis"
    question: string;        // 整体标题
  }
  ```
- [x] 消息 Schema 新增 `hidden` 字段到 `packages/shared/src/schemas/message.ts`
  ```typescript

  ```
