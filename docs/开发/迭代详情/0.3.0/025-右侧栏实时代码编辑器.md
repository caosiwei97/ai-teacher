> 状态：⬜ 待开始

## 目标

右侧栏新增代码编辑器 Tab，Agent 可通过 pushCode 工具推送代码到编辑器，用户可直接修改运行。这是竞品没有的独特功能。

## 核心设计

### 交互设计

```
右侧栏:
┌─────────────────────────┐
│ [路线图] [代码编辑器]     │  ← Tab 切换
├─────────────────────────┤
│ Python  ▼               │  ← 语言选择
│ ┌─────────────────────┐ │
│ │ def bubble_sort():  │ │  ← CodeMirror 编辑器（已有组件）
│ │   ...               │ │
│ └─────────────────────┘ │
│ [▶ 运行]    [🧹 清空]    │
│ ┌─────────────────────┐ │
│ │ 输出:                │ │  ← 运行结果
│ │ [1, 2, 3, 5, 8]     │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

### pushCode 工具

```typescript
{
  name: "pushCode",
  description: "推送代码到右侧编辑器面板，学生可直接修改运行",
  parameters: {
    code: string,        // 代码内容
    language: string,    // python | javascript | typescript | java | cpp
    instruction?: string // 给学生的说明（如"试试修改这里的条件"）
  }
}
```

### SSE 事件流

```
Agent pushCode 工具 → execute 返回 {success, code, language, instruction}
  → tutor-graph 发布 { type: "code-push", data: {code, language, instruction} }
  → use-chat-stream 接收 → 更新 codePanel state
  → 右侧栏 CodePanel 显示代码
```

## 关键注意事项

- pushCode 工具的 execute 不需要真的执行代码，只是把代码推送到前端——类似 renderUI 的模式
- 运行按钮由前端直接调 `useCodeExec` hook → Server sandbox API，不经过 Worker
- ThreeColumnLayout 的 right sidebar 渲染条件需调整：当前是 `nodes.length > 0` 才显示，需要改为始终显示（有 Tab 切换）
- Tutor prompt 的 `stopWhen: stepCountIs(3)` 可能需要调整到 4，因为多了一个 pushCode 工具调用
