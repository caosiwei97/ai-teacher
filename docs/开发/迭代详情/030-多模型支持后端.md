# 030 — 多模型支持：后端基础设施

> 状态：⬜ 待开始 | 分类：🔵 功能 | 优先级：P0 | 依赖：029
> 完成日期：—

## 目标

实现多 LLM 提供商支持的后端基础设施：用户可配置自己的 API Key + 选择模型，Worker/Server 动态创建 Provider 执行推理。

## 背景

当前系统硬编码 DeepSeek 单一提供商（`createDeepSeek` + `"deepseek-v4-flash"`），无法切换模型。用户希望使用 OpenAI、Anthropic、通义千问、Kimi、MiniMax、小米等提供商，并输入自己的 API Key。

### AI SDK v6 关键能力

- `createProviderRegistry()` — 统一注册多个 Provider，通过 `'providerId:modelName'` 字符串解析模型
- `createOpenAI({ baseURL })` — 覆盖所有 OpenAI 兼容协议（通义千问/Kimi/MiniMax/小米/自定义）
- `createAnthropic({ apiKey })` — Anthropic 原生协议
- `createDeepSeek({ apiKey })` — DeepSeek 原生（已安装）

## Checklist

### Phase 1：数据模型 + 迁移

- [ ] `packages/db/prisma/schema.prisma` 新增 `LlmConfig` model：
  ```prisma
  model LlmConfig {
    id            String   @id @default(cuid()) @db.Text
    userId        String   @db.Text
    user          User     @relation(fields: [userId], references: [id])
    provider      String   @db.Text    // "openai"|"anthropic"|"deepseek"|"qianwen"|"kimi"|"minimax"|"xiaomi"|"zhipu"|"custom"
    encryptedKey  String   @db.Text    // AES-256-GCM: iv:authTag:ciphertext（Base64）
    baseUrl       String?  @db.Text    // 自定义端点（预设提供商有默认值，custom/xiaomi 必填）
    defaultModel  String   @db.Text    // 用户选择的默认模型
    label         String?  @db.Text    // 用户自定义名称，如 "我的 OpenAI"
    isDefault     Boolean  @default(false)
    createdAt     DateTime @default(now())
    updatedAt     DateTime @updatedAt
    sessions     Session[]
    @@index([userId])
  }
  ```
