## 子任务① — 数据层 + 面试评分算法 TDD

### 算法设计（spec §4.2 + §9.2）

**难度动态调整**（spec §4.2）：连续 2 答对→升一档（easy→medium→hard 封顶）；连续 2 答错→降一档（hard→medium→easy 封底）；调整后 streak 归零（新难度重新计）。streak：正=连续答对，负=连续答错，0=无。

**纯函数**（落位 `packages/shared/src/services/interview-scoring.ts`）：

- `adjustDifficulty(currentDifficulty, streak, lastCorrect)` → `{ difficulty, streak }`
- `computeTotalScore(scores: number[])` → 0-100（平均分，无题则 0）
- `Difficulty` 类型 + `DIFFICULTY_LEVEL` 映射

**InterviewResult model**（spec §7）：

| 字段        | 类型                      | 默认        | 说明                                                             |
| ----------- | ------------------------- | ----------- | ---------------------------------------------------------------- |
| status      | InterviewStatus(enum)     | in_progress | in_progress/completed                                            |
| difficulty  | InterviewDifficulty(enum) | medium      | 当前难度 easy/medium/hard                                        |
| streak      | Int                       | 0           | 连续答对(+)/答错(-)                                              |
| totalScore  | Int                       | 0           | 复盘总评分 0-100                                                 |
| questionLog | Json?                     | null        | 每题记录 [{question,answer,score,isCorrect,difficulty,feedback}] |
| weakPoints  | Json?                     | null        | 薄弱点清单 string[]                                              |
| improvement | Text?                     | null        | 改进建议                                                         |

## 子任务② — 面试 Agent + API

### 设计（spec §4.1/§4.3 + §6.2）

**四阶段由面试官 prompt 驱动**：设定（开场白+第一题）→ 拷问（出题→听答→追问，绝不讲解）→ 每题即时评分（scoreAnswer tool）→ 复盘（finalizeInterview tool + renderUI interviewScore 评分卡）。

**难度状态跨轮持久化**：InterviewResult.difficulty/streak。chat-turn 每轮 `InterviewService.startOrGet` 取/建 in_progress 记录，注入 prompt（当前难度+streak+已答题数）。scoreAnswer 调 adjustDifficulty 更新。

**与复习关键区别**（spec §4.3，prompt 强制）：绝不讲解只追问；答错深挖"为什么"；高压基调；产物=纯对话+代码块/评分卡/复盘。
