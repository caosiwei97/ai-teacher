## 目标

兑现 spec §3 复习模式：基于间隔重复算法的智能推荐（今日到期清单）+ 提取练习三件套（抽认卡/回忆测验/错题本）+ 结束输出（记忆强度 + 下次复习时间 + 薄弱点清单）。核心：复习**不重讲概念**，让用户主动回忆——答对放行，答错才提示。

## 子任务① — 数据层 + 间隔重复算法 TDD

### 算法设计（spec §3.4 + §7.1 + §9.2）

**间隔策略**：复用 sigma 的 1d→2d→4d→8d→16d→32d（封顶 32d），持久化到 DB。

**纯函数**（落位 `packages/shared/src/services/spaced-repetition.ts`，server/worker 共用）：

- `isReviewDue(state, now)`：到期判断——`nextReviewAt` 为 null（未复习/逾期优先呈现，§7.1）或 ≤ now
- `computeNextInterval(currentIntervalDays, correct)`：答对翻倍（封顶 32d）/ 答错重置 1d
- `applyReviewResult(prev, correct, now)`：综合返回新记忆状态 `{ memoryStrength, lastReviewedAt, nextReviewAt, reviewInterval, trend }`
  - 记忆强度：答对 +0.15（封顶 1.0），答错 -0.3（floor 0.0）
  - 趋势 `trend`：强化（答对且强度上升）/ 维持（答对但已封顶）/ 衰退（答错）—— 对应 spec §3.3 结束输出
- `classifyMemory(strength)`：按绝对强度分档（仅供 UI 展示）

**字段默认值（§7.1，@default 兜底老数据，无需单独迁移脚本）**：

| 字段                  | 类型             | 默认     | 说明                         |
| --------------------- | ---------------- | -------- | ---------------------------- |
| `Node.memoryStrength` | Float            | 1.0      | 老节点视为刚掌握、强度满     |
| `Node.lastReviewedAt` | DateTime?        | null     | 未复习过                     |
| `Node.nextReviewAt`   | DateTime?        | null     | 进入复习模式按"逾期"优先呈现 |
| `Node.reviewInterval` | Int              | 1        | 首次复习间隔（天）           |
| `Session.activeMode`  | ActiveMode(enum) | learning | 老会话默认学习模式           |

> `activeMode` 用原生 enum（遵循 042/046 状态字段 Enum 化约定），与既有 `teachingMode`(warm/strict/interviewer) 区分——后者是学习阶段的对话风格，前者是学/固/验三阶段归属（spec §1 明确现有 interviewer≠面试模式）。

## 子任务② — 复习 Agent + API

### 设计（spec §3.2/§3.3 + §6.2/§6.3）

**双路径更新记忆强度（单一算法源）**：`applyReviewResult`（① 纯函数）是唯一算法源，两个入口调用：

- 抽认卡：学习者翻面自评 → UI（③）`POST /review/result` 提交（REST）
- 回忆测验：考官 agent 评分 → 调 `recordReviewResult` tool（worker）
- 两者都经 `ReviewService.submitResult`（shared，封装 applyReviewResult + prisma.update）

**chat-turn 分流**：worker 读 `session.activeMode`，`review` → 考官 prompt + 复习 tools（recordReviewResult + renderUI flashcard）；`learning` → 既有 tutor prompt + tutor tools（不变）。

**产物**：

- 抽认卡：`renderUI` 产 `flashcard` block（正面问题→翻面答案），③ 做翻面 UI
- 回忆测验：agent 文字提问 → 学习者作答 → agent 评分 → `recordReviewResult`
- 错题本：`GET /review/summary` 汇总薄弱点（memoryStrength 低 / 最近答错）
