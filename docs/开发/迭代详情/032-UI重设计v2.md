## 1. 目标

基于 ui-ux-pro-max 设计智能，全面重构 AI Teacher 的前端 UI，解决当前界面在**人体工程学**和**沉浸式体验**上的根本缺陷，打造让用户"想一直学下去"的专注学习环境。

---

## 2. 背景与问题

### 2.1 当前设计的问题（来自 UI 审计报告）

| 优先级 | 问题 | 影响 |
|--------|------|------|
| 🔴 关键 | Chat 输入框 `min-height: 24px`，有效输入区仅 ~8px | 人体工程学极差，长文本输入困难 |
| 🔴 关键 | 4 个 CSS 变量未定义（`chat-accent`, `chat-border`, `chat-muted`, `chat-hover`） | 部分组件样式断裂 |
| 🔴 关键 | 右侧栏在 `< 1024px` 屏幕完全不可见 | 平板/小笔记本用户无法使用路线图 |
| 🟡 高 | 30+ 处硬编码颜色值绕过设计系统 | 维护困难，主题切换不可能 |
| 🟡 高 | 输入框 `max-w-3xl` 与消息区域无限制，视觉错位 | 布局混乱 |
| 🟡 中 | 无统一动画标准（150ms ~ 500ms 不等） | 体验割裂 |
| 🟡 中 | 20+ 交互元素缺少 `focus-visible` 样式 | 可访问性失败 |
| ⚪ 低 | Geist 字体缺乏辨识度 | 品牌感弱 |

### 2.2 设计方向

采用 ui-ux-pro-max 推荐的 **"Deep Focus"** 设计哲学：
> 学者的圣殿 — 让外界消融，只剩下学习者与好奇心。

## 3. 设计系统规格

### 3.1 色彩体系

| Token | 旧值 | 新值 | WCAG |
|-------|------|------|------|
| `--color-background` | `#1a1918` | `#0f1117` | AAA |
| `--color-foreground` | `#e8e4dd` | `#e2e4e9` | AAA |
| `--color-primary` | `#d97706` | `#6366f1` | AA |
| `--color-accent` | `#f59e0b` | `#2dd4bf` | AAA |
| `--color-card` | `#252420` | `#161922` | — |
| `--color-muted` | `#2e2d2a` | `#1a1d2b` | — |
| `--color-border` | `#3a3835` | `#232738` | — |
| `--color-chat-user-bubble` | `#d97706` | `#1c2040` | AAA |
| `--color-chat-ai-bubble` | `#252420` | `#12151e` | AAA |

### 3.2 字体

- **正文**：Inter（替代 Geist）
- **代码**：JetBrains Mono
- **中文字体栈**：`"PingFang SC", "Microsoft YaHei", system-ui, sans-serif`

### 3.3 输入框人体工程学

| 属性 | 旧值 | 新值 |
|------|------|------|
| `min-height` | 24px | **56px** |
| `max-height` | 200px | **280px** |
| `padding` | py-2 (8px) | **p-4 (16px)** |
| `font-size` | 14px | **16px**（防 iOS 放大）|
| `border-radius` | 继承父级 | **12px** |
| Focus 环 | 1px 实线 | **3px 柔光光晕** |
| 发送按钮 | 32×32px | **40×40px** |

### 3.4 动画标准

| 场景 | 时长 | 缓动 |
|------|------|------|
| Hover | 150ms | ease |
| 侧边栏折叠 | 250ms | ease-out |
| 消息进入 | 300ms | `cubic-bezier(0.33, 1, 0.68, 1)` |
| 页面过渡 | 400ms | `cubic-bezier(0.16, 1, 0.3, 1)` |

### 3.5 响应式布局

| 断点 | 行为 |
|------|------|
| ≥1440px | 三栏全开（280px + 聊天 + 320px）|
| 1024–1439px | 左栏固定，右栏可折叠覆盖层 |
| 768–1023px | 双栏变覆盖层，聊天全宽 |
| <768px | 底部标签导航，全屏面板 |

## 6. 验证标准

1. **构建**：`pnpm build` 全量通过，无 TypeScript 错误
2. **E2E**：`npx playwright test` 24/24 通过
3. **人体工程学**：输入框最小 56px，focus 环 3px 柔光
4. **可访问性**：所有交互元素有 `focus-visible`，对比度 ≥ 4.5:1
5. **响应式**：1024px 以下右栏可访问，768px 以下移动端适配
6. **动画**：统一 150-400ms，支持 `prefers-reduced-motion`

## 7. 风险与回滚

| 风险 | 缓解措施 |
|------|----------|
| 颜色变量大规模变更导致组件样式断裂 | 逐文件修改 + LSP 诊断实时检查 |
| E2E 测试因 className 变更失败 | 同步更新 E2E 选择器 |
| 用户不适应新配色 | 保留设计系统规范文件，可随时调整 |
| 构建失败 | 每修改 3-5 个文件后运行一次 build |
