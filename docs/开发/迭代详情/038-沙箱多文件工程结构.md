# 迭代 038 — 沙箱多文件工程结构

> 状态：⬜ 待开始 | 优先级：P2 | 依赖：021, 037

## 背景

当前沙箱是单文件执行模式（OpenSandbox execd `/code` 端点），无法展示工程结构、多文件编辑。编程教学场景中，学生需要理解项目文件组织、模块化开发等概念。

## 目标

支持沙箱内的多文件工程结构，前端可查看和编辑多个文件。

## 方案调研（待确认）

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| A. OpenSandbox 文件管理 API | 利用 execd 的文件读写接口管理多文件 | 中 |
| B. E2B 沙箱 | 换用 E2B Code Interpreter（原生支持文件系统 + AI SDK） | 高 |
| C. Server 端文件管理 | Server 维护虚拟文件系统，按需写入沙箱执行 | 中 |

## Checklist
