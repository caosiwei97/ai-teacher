# 028 — SSE 事件扩展

> 状态：⬜ 待开始 | 分类：🟠 优化 | 优先级：P0 | 依赖：027
> 完成日期：—

## 目标

扩展 SSE 事件类型，实现路线图、会话状态、标题的实时推送。当前仅靠 `onFinish` 后 re-fetch session 数据，状态更新有延迟。

## 背景

当前 SSE 管道只有 6 种事件（text-delta、tool-call、tool-result、ui-blocks、code-push、done/error）。路线图和会话状态的变化需要等聊天回合结束后前端 re-fetch 才能反映。新增专用事件后，状态变化可实时推送到前端。

## Checklist

### Phase 1：Worker 端 SSE 事件新增

- [ ] 修改 `apps/worker/src/graphs/tutor-graph.ts`
  - 新增 `roadmap-updated` 事件：路线图生成/节点状态变化后发布
    ```typescript
    { type: "roadmap-updated", data: { nodes: RoadmapNode[] } }
    ```
  - 新增 `session-updated` 事件：会话标题/状态变化后发布
    ```typescript
    { type: "session-updated", data: { title?, learningStatus?, masteredNodes?, totalNodes? } }
    ```
  - 触发点：
    - `askQuestion` 诊断完成 → Agent 生成路线图后 → 发布 `roadmap-updated`
    - `assessMastery` 得分 ≥ 80 → 节点 mastered → 发布 `roadmap-updated` + `session-updated`
    - 首条消息设置标题 → 发布 `session-updated`

- [ ] 修改 `apps/worker/src/agent/tools/assess-mastery.ts`
  - 掌握通过后，查询最新节点状态，返回 `roadmapUpdate` 数据
  - 工具 execute 函数返回值新增 `{ roadmapUpdate: { nodes }, sessionUpdate: { masteredNodes, totalNodes } }`
  - `tutor-graph.ts` 在 tool-result 后检测这些字段并发布对应 SSE 事件

- [ ] 修改 `apps/worker/src/agent/tools/ask-question.ts`（或新建路线图触发逻辑）
  - 确保路线图生成后发布 `roadmap-updated` 事件

### Phase 2：前端 SSE 事件处理

- [ ] 修改 `apps/web/src/hooks/use-chat-stream.ts`
  - `AnnotationData` 新增字段：
    ```typescript
    roadmapUpdate?: { nodes: RoadmapNode[] };
    sessionUpdate?: { title?: string; learningStatus?: string; masteredNodes?: number; totalNodes?: number };
    ```
  - 新增 `roadmap-updated` 事件 → 存储到 `roadmapUpdate`
  - 新增 `session-updated` 事件 → 存储到 `sessionUpdate`

- [ ] 修改 `apps/web/src/app/learn/[sessionId]/page.tsx`
  - 监听 `roadmapUpdate` → 实时更新右侧栏路线图节点状态（无需等 re-fetch）
  - 监听 `sessionUpdate` → 实时更新 banner 标题和掌握度计数
  - 保留 `onFinish` re-fetch 作为兜底（确保最终一致性）
