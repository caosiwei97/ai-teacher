## 1. 产品概述

### 1.1 产品定位

AI 1v1 私教平台。用户上传学习资料（PDF/Markdown/网页），AI 基于资料生成知识图谱，通过苏格拉底式追问进行个性化教学。

### 1.2 核心差异化

| 特性 | 同类苏格拉底教学产品 | NotebookLM | AI Teacher（本产品） |
|------|-------|-----------|---------------------|
| 苏格拉底式追问 | ✅ | ❌ | ✅ |
| 用户上传资料 | ❌（预设课程） | ✅（仅总结） | ✅（基于资料教学） |
| 掌握度门控 | ✅ | ❌ | ✅ |
| 知识图谱可视化 | ✅ | ❌ | ✅ |
| 跨会话记忆 | ✅ | 部分 | ✅ |

### 1.3 用户故事

**US-001 上传学习资料**
> 作为学习者，我想上传 PDF/Markdown 文件或粘贴网页链接，让 AI 基于这些资料教我。

**US-002 生成学习路径**
> 作为学习者，上传资料后，系统自动生成可视化的知识图谱和学习路径，让我清楚知道要学什么。

**US-003 诊断摸底**
> 作为学习者，开始学习前 AI 先问几个问题，判断我当前的掌握程度，跳过我已经会的。

**US-004 苏格拉底式教学**
> 作为学习者，AI 不会直接告诉我答案，而是通过追问引导我自己思考，暴露我的理解漏洞。

**US-005 掌握度评估**
> 作为学习者，每个知识点都有明确的掌握度评分，达标后才能进入下一个知识点。

**US-006 学习回顾**
> 作为学习者，每个节点学完后有总结卡片（回顾表格+核心标签），方便我回顾。

**US-007 快问**
> 作为学习者，我可以随时选中内容提问，AI 结合当前学习上下文给我解答。

**US-008 跨会话记忆**
> 作为学习者，系统记住我的学习历史、易错点、学习偏好，下次回来不用从头开始。

## 2. 功能规格

### 2.1 学习资料管理

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 上传 PDF | 解析 PDF 内容，提取文本 | P0 |
| 上传 Markdown | 解析 Markdown 内容 | P0 |
| 粘贴网页链接 | 抓取网页内容 | P1 |
| 无资料模式 | 使用 AI 自有知识 | P0 |
| 资料管理 | 查看/删除已上传资料 | P1 |

### 2.2 知识图谱生成

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 自动生成节点 | 根据资料/主题分解为 5-15 个知识点 | P0 |
| 节点描述 | 每个节点有标题和简短描述 | P0 |
| 可视化展示 | 线性/树形节点图，状态标识 | P0 |
| 进度显示 | 已掌握/进行中/未开始 | P0 |

### 2.3 诊断评估

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 诊断问题 | 3-5 道选择题/简答题 | P0 |
| 水平判断 | 根据答案判断用户水平 | P0 |
| 起点定位 | 决定从哪个节点开始教学 | P0 |

### 2.4 苏格拉底式教学（核心）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 追问引擎 | 不直接给答案，追问暴露盲区 | P0 |
| 渐进提示 | 从最少到最多的提示层级 | P0 |
| 错误诊断 | 识别误解而非仅仅标记"错误" | P0 |
| 代码/示例展示 | 教学中使用代码块和示例 | P1 |
| 知识点追踪 | 记录擅长项和薄弱项 | P0 |
| 教学模式切换 | 严格/宽松 | P1 |

### 2.5 掌握度评估

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 实时评分 | 每轮对话后输出掌握度分数 | P0 |
| 门控机制 | ≥80% 解锁下一节点 | P0 |
| 评估卡片 | 节点总结+回顾表格+核心标签 | P0 |
| 自评校准 | 用户自评 vs AI 评分对比 | P1 |

### 2.6 会话与记忆

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 会话列表 | 学习中/新对话/已完成 | P0 |
| 学习者画像 | 跨会话记忆用户信息 | P0 |
| 易错点追踪 | 记录和回顾常见错误 | P1 |
| 会话恢复 | 断点续学 | P0 |

### 2.7 辅助功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 快问 | 选中内容直接提问 | P1 |
| AI 建议回复 | 帮助不知怎么答的用户 | P1 |
| 分享 | 分享学习会话 | P2 |

## 3. 页面结构

### 3.1 三栏布局（参考 同类苏格拉底教学产品）

```
┌──────────┬──────────────────────────────┬──────────────┐
│ 左侧栏   │ 中间对话区                    │ 右侧栏       │
│          │                              │              │
│ 学习资料  │  AI/用户 对话气泡             │ 学习路线图    │
│ ─────── │  知识讲解 + 代码示例           │              │
│ 学习中   │  评估卡片                     │ 进度 · 2 / 7 │
│ 新对话   │  输入框                       │ ✓ 节点1      │
│ 已完成   │  [教学模式] [快问] [AI建议]    │ ● 节点2      │
│          │                              │ ○ 节点3-7    │
└──────────┴──────────────────────────────┴──────────────┘
```

### 3.2 关键页面

| 页面 | 说明 |
|------|------|
| 首页/学习资料 | 上传资料或选择无资料模式，开始新学习 |
| 学习对话页 | 三栏布局，核心教学交互 |
| 会话历史 | 所有学习会话的列表 |

## 4. 数据模型（概要）

### 4.1 核心实体

```
User
  └── Source[] (学习资料)
  └── Session[] (学习会话)
        ├── Roadmap (知识图谱)
        │     └── Node[] (知识点节点)
        │           ├── status: not-started | in-progress | mastered
        │           └── masteryScore: 0-100
        ├── Message[] (对话消息)
        └── LearnerProfile (学习者画像)
```

### 4.2 消息类型

| 类型 | 说明 |
|------|------|
| tutor | AI 教学消息（追问/讲解/评估） |
| learner | 用户回答 |
| quiz | 诊断选择题 |
| quiz_response | 用户选择 |
| assessment | 节点评估卡片（结构化） |
| system | 系统消息（路线图生成、背景保存等） |
