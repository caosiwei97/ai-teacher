| # | 操作 | 期望结果 |

| 1 | 打开应用首页 | 页面加载完成，无报错 |

### E2E-A02 通过输入框创建新会话

**对应 同类苏格拉底教学产品 交互**：用户在首页输入主题，创建新的学习会话

#### 前置条件
- 应用已启动，在首页

#### 步骤

| # | 操作 | 期望结果 |
|---|------|----------|
| 1 | 在输入框输入 "React Hooks 的原理和使用" | 输入框显示输入内容，发送按钮变为可点击 |

- [ ] 输入主题后发送按钮可点击

- [ ] Banner 标题 = 用户输入的主题

### E2E-A04 诊断摸底卡片渲染与交互

**对应 同类苏格拉底教学产品 交互**：AI 返回含 3 个 Tab 的诊断选择卡片，每个 Tab 有 4-5 个梯度选项

#### 前置条件
- E2E-A02 完成，诊断卡片已出现

#### 步骤

| # | 操作 | 期望结果 |
|---|------|----------|
| 1 | 验证诊断卡片出现 | 对话区出现结构化诊断卡片 |
| 2 | 验证 Tab 导航 | 卡片有 3 个 Tab（如 React基础 / Hooks深入 / 学习目标） |
| 3 | 验证每个 Tab 的选项 | 每个 Tab 有 4-5 个梯度选项 + "其他（自由输入）" |
| 4 | 验证提交按钮初始状态 | 提交按钮 disabled |
| 5 | 在第一个 Tab 选择一个选项 | 该选项高亮，Tab 名称旁出现 ✓ |
| 6 | 切换到第二个 Tab，选择选项 | 第二个 Tab 也出现 ✓ |
| 7 | 切换到第三个 Tab，选择选项 | 第三个 Tab 也出现 ✓ |
| 8 | 验证提交按钮 | 所有 Tab 选完后，按钮变为可点击，文本为 "提交" |

#### 断言
- [ ] 诊断卡片有 3 个 Tab
- [ ] 每个 Tab 有 4-5 个选项 + "其他（自由输入）"
- [ ] 选项有明确的水平区分度
- [ ] 未全部回答时提交按钮 disabled
- [ ] 已回答的 Tab 有 ✓ 标记
- [ ] 全部回答后提交按钮可点击
