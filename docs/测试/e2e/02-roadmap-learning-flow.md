# ## B. 路线图与学习流程

> [← 返回测试矩阵总览](./README.md)

### E2E-B01 路线图卡片自动生成

**对应 同类苏格拉底教学产品 交互**：诊断提交后，AI 自动生成 5-9 个节点的路线图卡片

#### 前置条件
- E2E-A05 完成

#### 步骤

| # | 操作 | 期望结果 |
|---|------|----------|
| 1 | 等待路线图卡片生成 | 对话中出现路线图卡片，标题为 "学习路线: {主题}" |
| 2 | 验证节点数量 | 路线图包含 5-9 个节点卡片 |
| 3 | 验证节点卡片内容 | 每个节点有标题（简短）+ 描述（一句话） |
| 4 | 验证右侧栏同步 | 右侧栏 "路线图" Tab 显示相同的节点列表 |
| 5 | 验证进度初始化 | 右侧栏进度显示 "进度 . 0 / N"，Banner 显示 "掌握 0/N" |
| 6 | 验证教学自动开始 | 路线图生成后，AI 自动开始第一个节点的教学（提问式） |

#### 断言
- [ ] 路线图卡片有 "学习路线: {主题}" 标题
- [ ] 节点数在 5-9 范围
- [ ] 每个节点有标题 + 描述
- [ ] 右侧栏节点列表与对话中路线图一致
- [ ] 进度初始化为 0/N
- [ ] 自动开始第一个节点教学

### E2E-B02 节点解锁与推进

**对应 同类苏格拉底教学产品 交互**：评估通过后点击 "下一节" 按钮，进入下一个知识节点

#### 前置条件
- E2E-D02 完成，评估卡片已显示

#### 步骤

| # | 操作 | 期望结果 |
|---|------|----------|
| 1 | 点击评估卡片底部 "下一节: {标题}" 按钮 | 页面滚动到底部 |
| 2 | 验证新节点教学 | AI 开始第二个节点的教学（新主题 + 提问） |
| 3 | 验证 Banner 进度 | 显示 "掌握 1/N" |
| 4 | 验证右侧栏 | 进度更新为 "进度 . 1 / N"，第一个节点 ✓，第二个节点高亮 |
| 5 | 验证左侧栏 | 会话卡片进度显示 "1/N" |

#### 断言
- [ ] 点击 "下一节" 后 AI 自动开始新节点教学
- [ ] Banner 进度更新
- [ ] 右侧栏进度更新，前一节点 ✓，当前节点高亮
- [ ] 左侧栏进度更新
- [ ] 新节点内容与旧节点不重复

### E2E-B03 进度同步（Banner + 右侧栏 + 左侧栏）

**对应 同类苏格拉底教学产品 交互**：三处进度显示始终保持一致

#### 前置条件
- E2E-B02 完成

#### 步骤

| # | 操作 | 期望结果 |
|---|------|----------|
| 1 | 读取 Banner "掌握 X/N" | 记录 X 和 N |
| 2 | 读取右侧栏 "进度 . X / N" | 与 Banner 一致 |
| 3 | 读取左侧栏会话卡片进度 "X/N" | 与 Banner 一致 |
| 4 | 完成一个新节点的评估 | 三处进度同步 +1 |

#### 断言
- [ ] Banner、右侧栏、左侧栏三处进度数值完全一致
- [ ] 节点完成后三处同步更新

### E2E-B04 会话恢复（断点续学）

**对应 同类苏格拉底教学产品 交互**：刷新页面或关闭重开后，学习进度和对话完整保留

#### 前置条件
- E2E-B01 完成，至少进行了一部分教学对话

#### 步骤

| # | 操作 | 期望结果 |
|---|------|----------|
| 1 | 刷新页面（F5） | 页面重新加载 |
| 2 | 验证对话历史 | 所有消息完整保留，滚动到最新消息 |
| 3 | 验证进度 | Banner 和右侧栏进度与刷新前一致 |
| 4 | 继续输入 | 输入框可输入，AI 记得之前的上下文 |
| 5 | 关闭浏览器，重新打开应用 | 点击左侧栏对应会话 |
| 6 | 验证跨 session 恢复 | 会话完整恢复（消息 + 进度 + 路线图状态） |

#### 断言
- [ ] 刷新后对话历史完整
- [ ] 进度状态保留
- [ ] AI 保持上下文连贯
- [ ] 关闭重开后仍可恢复
