## ADR-001: 智谱 GLM 作为默认 LLM

**日期**：2026-05-08
**状态**：已采纳

### 决策

使用智谱 GLM（OpenAI 兼容接口）作为默认 LLM，模型 `glm-4-flash`。

### 背景

初期规划使用 OpenAI GPT-4o / Claude，实际开发时因网络和成本考虑改用智谱。

### 理由

- 国内直连，无需代理
- OpenAI 兼容接口，可直接使用 AI SDK 的 `createOpenAI` provider
- `glm-4-flash` 性价比高，适合高频教学对话
- 可用模型：`glm-4-flash`（默认）、`glm-4.5`、`glm-5`

### 影响

- `OPENAI_BASE_URL` 指向智谱 API
- 所有 Agent（Tutor、Diagnostic、Roadmap）均使用同一 provider
- 可通过修改 `.env` 切换到其他 OpenAI 兼容接口

## ADR-002: pnpm workspaces 替代 npm workspaces

**日期**：2026-05-08
**状态**：已采纳

### 决策

使用 pnpm workspaces 管理 monorepo（apps/web + apps/worker + packages/*）。

### 理由

- 用户建议
- pnpm 磁盘占用更小（content-addressable store）
- 严格的依赖隔离，减少幽灵依赖
- `.npmrc` 中 `shamefully-hoist=true` 确保 Next.js/Prisma 依赖正确解析

## ADR-003: 非标准端口

**日期**：2026-05-08
**状态**：已采纳

### 决策

所有服务使用非标准端口避免本地冲突。

| 服务 | 端口 | 标准端口 |
|------|------|----------|
| Web | 38421 | 3000 |
| Worker | 38422 | 4000 |
| PostgreSQL | 25432 | 5432 |
| Redis | 26379 | 6379 |
| MinIO API | 29000 | 9000 |
| MinIO Console | 29001 | 9001 |

### 理由

- 开发者本地可能已有标准端口服务在运行
- 避免端口冲突导致启动失败

## ADR-004: .env 通过 symlink 共享

**日期**：2026-05-08
**状态**：已采纳

### 决策

根目录 `.env` 通过 symlink 共享给子项目：`apps/web/.env → ../../.env`。

### 理由

- Next.js 子项目无法直接读取父目录的 `.env`
- Prisma 需要通过 `dotenv-cli` 读取根目录 `.env`
- symlink 方案零依赖，所有子项目共享同一份配置

## ADR-005: AI SDK streamText + tool calling

**日期**：2026-05-08
**状态**：已采纳

### 决策

Tutor Agent 使用 AI SDK v4 的 `streamText` + `tool calling` 实现 Agent Loop。

### 理由

- `streamText` 支持 SSE 流式输出，用户体验好
- `tool calling` 让 LLM 在对话中调用结构化工具（assessMastery 等）
- AI SDK 提供统一的 provider 接口，方便切换 LLM
- `generateObject` + Zod Schema 约束 Diagnostic/Roadmap Agent 的输出格式

## ADR-006: Tailwind CSS 4 + CSS 变量设计系统

**日期**：2026-05-08
**状态**：已采纳

### 决策

使用 Tailwind CSS 4 的 `@theme` 定义设计系统，颜色用 CSS 变量而非硬编码。

### 理由

- Tailwind CSS 4 原生支持 CSS 变量，无需 `tailwind.config.js`
- 暖色教育主题（stone + amber）通过 CSS 变量统一管理
- 组件使用语义化变量（`var(--sidebar-bg)` 而非 `bg-slate-900`），方便全局调整

### 约束

- PostCSS 必须用 `module.exports`（不用 `export default`），与 AI SDK 有兼容性问题
- 颜色必须用 `@theme` CSS 变量，不用硬编码色值

## ADR-007: Roadmap Agent LLM 生成 + fallback

**日期**：2026-05-08
**状态**：已采纳

### 决策

知识图谱通过 LLM 动态生成，失败时 fallback 到通用 5 节点模板。

### 理由

- LLM 生成的节点更贴合主题，粒度更合理
- fallback 保证即使 LLM 不可用也能正常创建会话
- `generateObject` + Zod Schema 约束节点数量在 5-15 之间

## ADR-008: Web 直接调用 Worker（跨 workspace import）

**日期**：2026-05-08
**状态**：已采纳

### 决策

Web 的 API Routes 直接 import Worker 的 Agent 函数（如 `streamTutorResponse`），而非通过 HTTP 调用。

### 理由

- 开发阶段简单直接，无需额外的进程间通信
- Next.js webpack 支持跨 workspace import
- 流式响应直接通过 `result.toDataStreamResponse()` 返回

### 风险

- 耦合较紧，后续如果 Worker 独立部署需要重构为 HTTP 调用

## ADR-009: Agent 引擎重构 — BaseAgent + 工具副作用内化

**日期**：2026-05-09
**状态**：已采纳

### 决策

重构 Agent 架构：创建 `BaseAgent` 抽象基类，统一 Provider 创建，将工具副作用从 API 层移入工具 execute 函数。

### 背景

原架构问题：
- 5 个 tool 的 execute 只是 passthrough（返回 `{ success: true }`），副作用散落在 `chat/route.ts` 的 `persistChatTurn` 中（293 行上帝函数）
- 3 个 Agent 各自创建 provider，无共享基础设施
- 无上下文管理，长对话必然崩溃
- 无错误恢复，LLM 调用失败直接 crash

### 理由

- `BaseAgent` 统一 provider 管理、model 获取、重试/fallback 逻辑
- 工具副作用内化后，`chat/route.ts` 从 293 行降至 166 行
- `NodeService` / `MessageService` / `ProfileService` 封装业务逻辑
- `createTutorTools()` 工厂函数使工具可访问 service 层

### 参考

- [Agent引擎优化方案](Agent引擎优化方案.md)
- 参考仓库：Open Agents (Vercel) → ToolLoopAgent + prepareCall/prepareStep
- 参考仓库：Pi Framework → beforeToolCall/afterToolCall 钩子

## ADR-010: 消息截断 + Token 预算估算

**日期**：2026-05-09
**状态**：已采纳

### 决策

实现 `transformMessages()` 消息截断策略和 `estimateTokens()` Token 预算估算。

### 理由

- 智谱 `glm-4-flash` 上下文窗口 128K tokens，但长对话仍需截断
- 保护策略：保护首尾（系统上下文通过 system prompt 传递 + 最近 10 轮），压缩中间
- 单条消息超过 2000 字符时截断
- Token 估算用简单字符计数（中文 1.5 token/字，英文 0.25 token/字）

## ADR-011: LLM 重试 + Fallback 模型链

**日期**：2026-05-09
**状态**：已采纳

### 决策

在 `BaseAgent` 中实现 `executeWithRetry()`（指数退避 1s/2s/4s）和 `executeWithFallback()`（主模型→fallback）。

### 理由

- 智谱 API 偶尔返回 429/500/503，重试可自动恢复
- 指数退避避免加剧限流
- Fallback 模型链提供降级能力（主模型不可用时切换）
- 流式响应不适合重试（流已返回客户端），重试仅用于 `generateObject` 调用（Roadmap/Diagnostic Agent）

## ADR-012: 学习者画像 + 跨会话记忆

**日期**：2026-05-09
**状态**：已采纳

### 决策

会话完成时自动更新 `LearnerProfile`，新会话创建时注入画像到 Agent prompt。

### 理由

- Prisma Schema 已有 `LearnerProfile` model（strengths/weaknesses/sessionsSummary）
- `ProfileService.updateAfterSession()` 在 PATCH status=completed 时触发
- `buildLearnerProfile()` 在 `chat/route.ts` 中已实现画像→文本转换
- 保留最近 10 次会话摘要，避免无限增长
