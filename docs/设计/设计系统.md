# AI Teacher — Design System v2: "Deep Focus"

> Version 2.0 · 2026-05-11

---

## 1. Design Philosophy

AI Teacher's visual language draws from the concept of a **"scholar's sanctum"** — a space so calm and enveloping that the outside world dissolves, leaving only the learner and their curiosity. We pair deep, cool-neutral backgrounds with muted indigo and teal accents that evoke intellectual depth without stimulating fatigue. Every surface recedes; every interactive element breathes. The result is an interface that feels less like software and more like a private study carved from obsidian and moonlight — premium, unhurried, and magnetically focused.

  --font-mono:  var(--font-mono), "Cascadia Code", "Fira Code", ui-monospace, monospace;

### 3.2 Type Scale

| Token | Size | Line Height | Weight | Usage |
|-------|------|-------------|--------|-------|
| `text-xs` | 12px / 0.75rem | 1.5 (18px) | 400 | Timestamps, badges, footnotes |
| `text-sm` | 14px / 0.875rem | 1.5 (21px) | 400 | Secondary labels, sidebar items, meta |
| `text-base` | 16px / 1rem | 1.625 (26px) | 400 | Body text, chat messages — optimized for reading |
| `text-lg` | 18px / 1.125rem | 1.556 (28px) | 500 | Section headers in chat, card titles |
| `text-xl` | 20px / 1.25rem | 1.4 (28px) | 600 | Page-level headings |
| `text-2xl` | 24px / 1.5rem | 1.333 (32px) | 600 | Hero titles, empty states |
| `text-3xl` | 32px / 2rem | 1.25 (40px) | 700 | Landing/splash only |

### 3.3 Reading Optimization

- **Chat message max-width**: `680px` (optimal 60–75 characters per line at 16px)
- **Chat line-height**: `1.625` (generous for long-form reading)
- **Letter-spacing**: `0` for body, `-0.01em` for headings ≥ 20px
- **Paragraph spacing in AI responses**: `1rem` (margin-bottom on `<p>`)
- **Code block font size**: 14px (one step down from body)

## 5. Component Styling Rules

### 5.1 Chat Input — "The Composer"

The most-touched element. Must feel spacious and inviting.

```
┌──────────────────────────────────────────────────┐
│ ┌──────────────────────────────────────────────┐ │
│ │  [textarea]                                  │ │  ← min-height: 56px
│ │  Write your answer…                          │ │     max-height: 280px
│ │                                              │ │     padding: 16px 48px 16px 16px
│ └──────────────────────────────────────────────┘ │
│                                        [Send ▶] │  ← 40×40 button, bottom-right
└──────────────────────────────────────────────────┘
   ↑ outer wrapper: padding 16px, border-top 1px --border
```

| Property | Value | Rationale |
|----------|-------|-----------|
| `min-height` | 56px (3.5rem) | Comfortable single-line, not cramped |
| `max-height` | 280px (17.5rem) | ~10 lines visible before scroll |
| `padding` | 16px 48px 16px 16px | Right padding reserves space for send button |
| `font-size` | 16px | Prevents iOS zoom, matches body |
| `background` | `var(--chat-input-bg)` | |
| `border` | 1px solid `var(--chat-input-border)` | |
| `border-radius` | 12px | Slightly rounder than default `--radius` for warmth |
| `focus border` | `var(--chat-input-focus)` | Indigo glow on focus |
| `focus ring` | `0 0 0 3px color-mix(in srgb, var(--ring) 25%, transparent)` | Soft outer glow |
| `resize` | `none` | Auto-grow via JS instead |
| `transition` | `border-color 200ms ease, box-shadow 200ms ease` | |

### 5.2 Buttons

| Variant | Background | Text | Border | Hover |
|---------|-----------|------|--------|-------|
| **Primary** | `--primary` | `--primary-foreground` | none | `--primary-hover`, translateY(-1px), shadow-md |
| **Secondary** | `--secondary` | `--secondary-foreground` | 1px `--border` | `--secondary-hover` |
| **Ghost** | transparent | `--muted-foreground` | none | `--secondary` bg |
| **Destructive** | `--destructive` | `--destructive-foreground` | none | `--destructive-hover` |
| **Icon** | transparent | `--muted-foreground` | none | `--secondary` bg, text → `--foreground` |

All buttons: `min-height: 40px`, `min-width: 40px`, `border-radius: var(--radius)`, `font-weight: 500`, `transition: all 150ms ease`.

### 5.3 Cards & Surfaces

```css
.card {
  background: var(--color-card);
  border: 1px solid var(--color-border);
  border-radius: var(--radius);
  padding: 16px;  /* or 24px for large cards */
}
```

- **Elevation**: No box-shadow in dark mode. Use border opacity to suggest depth.
- **Hover state** (for interactive cards): `border-color: var(--border-strong)`, `transition: 150ms`.

## 7. Ergonomics Guidelines

### 7.1 Touch Targets & Interactive Elements

| Element | Minimum Size | Rationale |
|---------|-------------|-----------|
| Buttons | 40×40px (desktop), 44×44px (mobile) | WCAG 2.5.8 target size |
| Sidebar items | 44px tall | Comfortable click/tap |
| Icon buttons | 40×40px with 8px inner padding on icon | Generous hit area |
| Links in text | 24px line-height minimum | Finger-friendly in touch context |

### 7.2 Chat Input Ergonomics

- **Auto-grow**: textarea grows with content from 56px → 280px, then scrolls
- **Submit**: Enter sends, Shift+Enter inserts newline (display hint below input)
- **Focus trap**: On page load, auto-focus the input after initial content loads
- **Character counter**: Show at 80% of max length (if applicable), muted text
- **Sticky position**: Input bar fixed at bottom of chat column, never scrolls away
- **Mobile**: Input expands to full width, send button stays accessible

### 7.3 Focus States

Every interactive element must have a visible focus indicator:

```css
:focus-visible {
  outline: none;
  box-shadow: 0 0 0 2px var(--color-background),
              0 0 0 4px var(--color-ring);
}
```

- 2px gap (background color) + 2px ring (primary) = clear on all backgrounds
- Never use `outline: none` without a replacement indicator

### 7.4 Scrollbar Styling

```css
::-webkit-scrollbar {
  width: 6px;
}
::-webkit-scrollbar-track {
  background: transparent;
}
::-webkit-scrollbar-thumb {
  background: var(--color-border);
  border-radius: 3px;
}
::-webkit-scrollbar-thumb:hover {
  background: var(--color-border-strong);
}
```

### 7.5 Layout Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| ≥ 1440px | Full 3-column layout |
| 1024–1439px | Left sidebar collapsible, right sidebar overlay |
| 768–1023px | Both sidebars as overlays, chat full width |
| < 768px | Mobile: bottom tab nav, full-screen panels |

## 8. Anti-Patterns to Avoid

| ❌ Don't | ✅ Do Instead | Why |
|----------|--------------|-----|
| Pure white text (#ffffff) on dark bg | Use #e2e4e9 (soft white) | Pure white causes eye strain in dark mode |
| Primary color on large surfaces | Primary only for interactive elements & small accents | Indigo is for action, not decoration |
| Drop shadows in dark mode | Use border opacity (1px solid border tokens) | Shadows are invisible on dark backgrounds |
| Rounded corners > 16px on containers | Max 12px for cards, 10px for buttons | Over-rounding looks juvenile |
| Animated gradients or particle effects | Subtle, single-property transitions | Motion sickness, performance drain |
| Auto-playing sound or haptics | Silent by default, user-opt-in | Jarring during study sessions |
| Text smaller than 13px for readable content | 14px minimum for any content text | Readability > information density |
| Bright colors for status indicators | Use opacity variants of accent/destructive | Neon dots distract from content |
| Collapsing chat input to single-line | Keep 56px minimum height always | Users type long Socratic responses |
| Using `opacity` for disabled states alone | Use `opacity: 0.5` + `cursor: not-allowed` + desaturated colors | Opacity alone is ambiguous |
| Placeholder text as labels | Always use visible labels or floating labels | Placeholders disappear on input |
| Custom fonts for body text | Inter (or system) — proven readability at all sizes | Novel fonts slow reading speed |
| Inline styles or magic numbers | Use design tokens from this spec | Consistency and maintainability |
| `z-index` arms race (999, 9999) | Layering system: base(0), sidebar(10), popover(20), modal(30), toast(40) | Predictable stacking |
