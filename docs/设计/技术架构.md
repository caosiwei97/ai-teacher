## 1. 技术栈

| 层 | 技术 | 理由 |
|---|------|------|
| Runtime | Node.js 20 LTS, TypeScript strict | 生态成熟，AI SDK 原生支持 |
| Web | Next.js 15 (App Router), React 19 | SSR + API Routes 一体化 |
| UI | shadcn/ui + Tailwind CSS 4 | 快速开发，护眼暖色暗色设计系统（4000K 色温） |
| 编辑器 | CodeMirror 6 | 嵌入式代码编辑器，语法高亮 + 暖色自定义主题 |
| 公式 | KaTeX | LaTeX 数学公式渲染 |
| Agent | AI SDK v4 (streamText + tool calling) | 成熟的 Agent Loop，流式输出 |
| LLM | 智谱 GLM（OpenAI 兼容接口） | 国内可用，性价比高，默认 `glm-5-turbo` |
| DB | PostgreSQL 16 + pgvector + Prisma ORM | 结构化数据 + 向量检索（RAG 预留） |
| Cache | Redis 7 | 会话状态 + 对话缓存（预留） |
| 存储 | MinIO (S3 兼容) | 用户上传的文件存储（预留） |
| 沙箱 | Judge0 (自部署) | 60+ 语言安全代码执行，Docker 容器隔离 |
| 包管理 | pnpm workspaces | Monorepo 依赖管理 |

## 4. 数据模型

### 4.1 Prisma Schema（实际实现）

```prisma
model User {
  id        String          @id @default(cuid()) @db.Text
  name      String          @db.Text
  sessions  Session[]
  sources   Source[]
  profile   LearnerProfile?
  createdAt DateTime        @default(now())
}

model Source {
  id        String    @id @default(cuid()) @db.Text
  userId    String    @db.Text
  user      User      @relation(fields: [userId], references: [id])
  title     String    @db.Text
  type      String    @db.Text          // pdf | markdown | url
  content   String?   @db.Text          // 解析后的文本
  fileUrl   String?   @db.Text          // MinIO 文件 URL
  checksum  String?   @db.Text          // 文件哈希
  sessions  Session[]
  createdAt DateTime  @default(now())
}

model Session {
  id        String    @id @default(cuid()) @db.Text
  userId    String    @db.Text
  user      User      @relation(fields: [userId], references: [id])
  topic     String    @db.Text
  sourceId  String?   @db.Text
  source    Source?   @relation(fields: [sourceId], references: [id])
  status    String    @default("diagnosing") @db.Text  // diagnosing | active | completed | archived
  summary   Json?     // StructuredSummary — 上下文压缩后的结构化摘要
  roadmap   Roadmap?
  messages  Message[]
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
}

model Roadmap {
  id        String  @id @default(cuid()) @db.Text
  sessionId String  @unique @db.Text
  session   Session @relation(fields: [sessionId], references: [id])
  nodes     Node[]
  version   Int     @default(1)
}

model Node {
  id           String   @id @default(cuid()) @db.Text
  roadmapId    String   @db.Text
  roadmap      Roadmap  @relation(fields: [roadmapId], references: [id])
  index        Int      // 排序
  title        String   @db.Text
  description  String   @db.Text
  status       String   @default("not-started") @db.Text  // not-started | in-progress | mastered
  masteryScore Int      @default(0)                         // 0-100
  reviewLog    Json?    // 评估回顾数据
  masteredAt   DateTime?
}

model Message {
  id        String   @id @default(cuid()) @db.Text
  sessionId String   @db.Text
  session   Session  @relation(fields: [sessionId], references: [id])
  role      String   @db.Text    // tutor | learner | system
  type      String   @db.Text    // text | quiz | quiz_response | assessment | system
  content   String   @db.Text
  metadata  Json?    // 结构化数据（工具调用结果、评估卡片等）
  createdAt DateTime @default(now())
}

model Checkpoint {
  id        String   @id @default(cuid()) @db.Text
  sessionId String   @db.Text
  session   Session  @relation(fields: [sessionId], references: [id])
  graphNode String   @db.Text   // StateGraph 节点名（prepare_context / agent_loop / post_process）
  state     Json               // 序列化的 TutorState
  createdAt DateTime @default(now())

  @@index([sessionId, createdAt])
}

model LearnerProfile {
  id                   String   @id @default(cuid()) @db.Text
  userId               String   @unique @db.Text
  user                 User     @relation(fields: [userId], references: [id])
  learningStyle        Json?    // 学习偏好
  strengths            Json?    // 擅长领域
  weaknesses           Json?    // 薄弱领域
  misconceptionPatterns Json?   // 常见误解模式
  sessionsSummary      Json?    // 历史学习摘要
  updatedAt            DateTime @updatedAt
}
```

### 5.1 StateGraph 工作流（迭代 018）

```typescript
// TutorGraph: 3 个节点 + 1 条条件边
const tutorGraph = new StateGraph<TutorState>()
  .setEntryPoint("prepare_context")
  .addNode("prepare_context", ...)  // ContextManager: 消息分层 + Token 估算 + Compaction
  .addNode("agent_loop", ...)       // streamText + tool calling
  .addNode("post_process", ...)     // checkpoint 保存
  .addEdge("prepare_context", "agent_loop")
  .addConditionalEdge("agent_loop", (state) =>
    state.needsFollowUp ? "prepare_context" : "post_process")
  .addEdge("post_process", END);

// 每个节点执行前自动保存 Checkpoint（可断点恢复）
// agent_loop 内部使用 ToolRegistry.toAiSdkTools(ctx) 获取 DI 工具
```

## 5.4 A2UI 框架（迭代 020）

AI-to-UI（A2UI）是消息渲染的抽象层，将 Agent 输出映射为结构化 UIBlock，前端通过 Registry 自动选择对应组件渲染。

### UIBlock 类型

| type | 组件 | 说明 |
|------|------|------|
| `text` | TextBlock | Markdown 文本（含代码块、行内代码） |
| `assessment` | AssessmentBlock | 学习评估卡片（包装现有 AssessmentCard） |
| `quiz` | QuizPanel | 交互式测验（可点击选项 + 即时反馈） |
| `code-result` | CodeResultBlock | 代码运行结果（代码 + stdout/stderr + 退出码） |
| `formula` | FormulaDisplay | LaTeX 公式（KaTeX 渲染） |
| `diagram` | DiagramRenderer | 图表（预留占位） |

### 渲染流程

```
Message → 检查 metadata.uiBlocks
  ├── 有 uiBlocks → 遍历 blocks，Registry 查找组件渲染
  └── 无 uiBlocks → 降级为单个 TextBlock（向后兼容）
```

### 依赖

| 包 | 用途 |
|----|------|
| `codemirror` + `@codemirror/*` | CodeMirror 6 代码编辑器（语法高亮 + 暖色主题） |
| `katex` | LaTeX 公式渲染 |

### 10.1 目标架构：三层分离

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户浏览器                                 │
│  Next.js SPA (UI Only, 无 API Routes)                           │
│  - Chat 对话 + Code Editor (CodeMirror 6)                       │
│  - Roadmap / Diagnostic / UIBlock Renderer                      │
└────────┬──────────────────────────────┬─────────────────────────┘
         │ HTTP/SSE                     │ HTTP
         ▼                              ▼
┌─────────────────┐           ┌──────────────────────────────────┐
│  Hono API       │           │  Sandbox Service                  │
│  :38421         │           │  :38423 (Judge0)                  │
│  - 路由代理      │           │  - 代码执行 (Docker 容器隔离)       │
│  - Auth         │           │  - 多语言支持                      │
│  - Rate Limit   │           │  - 资源限制 (CPU/MEM/TIME)         │
└────────┬────────┘           └──────────────────────────────────┘
         │ BullMQ Queue + Redis Pub/Sub
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  Worker :38422                                                  │
│  - Agent Engine (StateGraph + Tool Registry)                    │
│  - Background Jobs (RAG 索引, Profile 聚合)                     │
│  - Sandbox Client (调用 Judge0)                                 │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
   PostgreSQL :25432  +  Redis :26379
```

### 10.2 关键技术决策

| 决策 | 选型 | 理由 |
|------|------|------|
| 后端框架 | Hono（非 NestJS） | 轻量、函数式、AI SDK 原生兼容 |
| Agent 引擎 | 自实现 StateGraph（非 LangGraph.js） | 借鉴 LangGraph 概念，AI SDK 原生 |
| 消息队列 | BullMQ + Redis Pub/Sub | 可靠持久化 + 实时流式推送 |
| A2UI 方案 | Tool-based UI + Component Registry | 安全、可调试、类型安全 |
| 代码编辑器 | CodeMirror 6（非 Monaco） | 轻量 200KB、多实例、移动端友好 |
| 代码沙箱 | Judge0 | 自部署、60+ 语言、Docker 隔离 |
| 技术栈升级 | AI SDK v5 + Zod 4 | 无历史负担，一步到位 |
