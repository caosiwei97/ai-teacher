## 1. 技术栈

| 层 | 技术 | 理由 |
|---|------|------|
| Runtime | Node.js 20 LTS, TypeScript strict | 生态成熟，AI SDK 原生支持 |
| Web | Vite + React 19 (SPA) | 轻量构建，纯客户端渲染 |
| UI | shadcn/ui + Tailwind CSS 4 | 快速开发，护眼暖色暗色设计系统（4000K 色温） |
| 编辑器 | Monaco Editor (@monaco-editor/react) | IDE 级代码编辑器，多语言支持 + Catppuccin 暖色主题 |
| 公式 | KaTeX | LaTeX 数学公式渲染 |
| Agent | AI SDK v6 (streamText + tool calling) | 成熟的 Agent Loop，流式输出 |
| LLM | DeepSeek（@ai-sdk/deepseek 原生 provider） | 国内可用，性价比高，默认 `deepseek-v4-flash` |
| DB | PostgreSQL 16 + pgvector + Prisma ORM | 结构化数据 + 向量检索（RAG 预留） |
| Cache | Redis 7 | 会话状态 + 对话缓存（预留） |
| 存储 | MinIO (S3 兼容) | 用户上传的文件存储（预留） |
| 沙箱 | OpenSandbox (Judge0 API 兼容) | Docker 容器隔离代码执行，文件系统 + PTY 终端 |
| 包管理 | pnpm workspaces | Monorepo 依赖管理 |

### 5.1 Agent Loop（run-agent-loop.ts）

```typescript
// 主 Agent 循环：手动多步迭代
// 每步调用 streamText(maxSteps=1) 执行一轮 tool calling
// 如果有工具调用 → 处理工具结果 → 继续下一轮
// 如果无工具调用 → 循环结束
async function runAgentLoop(params: AgentLoopParams) {
  for (let step = 0; step < MAX_STEPS; step++) {
    const result = await streamText({
      model: providerRegistry.getModel(llmConfig),
      system: buildSystemPrompt(promptSections),
      messages: contextManager.buildMessages(session),
      tools: createTutorTools({ sessionId, userId, subagents }),
      maxSteps: 1,  // 每步只执行一轮
    });

    // 流式推送 SSE 事件
    publishSSEEvents(result);

    // 无工具调用 → 结束循环
    if (!result.toolCalls?.length) break;

    // 处理工具结果
    await processToolResults(result.toolCalls);
  }
}
```

### 5.1 StateGraph 工作流（迭代 018）

  .addNode("prepare_context", ...)  // ContextManager: 消息分层 + Token 估算 + Compaction

### 5.4 多 Agent 协作 + Subagent（迭代 022）

主 Agent（TutorAgent）可通过 `delegateTask` 工具将任务委派给专业子 Agent。每个子 Agent 有独立的 system prompt、工具集、模型和步数限制，执行结果经 `toModelOutput()` 过滤后返回摘要，防止上下文污染。

```typescript
// SubagentRegistry — apps/worker/src/agent/subagents/index.ts
interface SubagentDefinition {
  name: string;
  description: string;
  systemPrompt: string;
  tools: string[];
  maxSteps: number;
  model?: string;        // 可指定模型
  toModelOutput: (result: AgentResult) => string;
}

// 注册的子 Agent
const registry = createSubagentRegistry();
// - assessment: 出练习题、评估答案、学习报告（maxSteps: 3, model: deepseek-v4-flash）
// - research: 搜索教学资料、补充参考（maxSteps: 5, model: deepseek-v4-flash）
```

```
TutorAgent (主)
  │
  ├── delegateTask({ agent: "assessment", task: "出 3 道 React Hooks 练习题" })
  │     └── AssessmentAgent (独立 streamText)
  │           ├── systemPrompt: 专业出题评估 Agent
  │           ├── tools: [assessMastery, generateAssessment]
  │           ├── maxSteps: 3
  │           └── toModelOutput → 摘要（≤300 字）
  │
  └── delegateTask({ agent: "research", task: "搜索 React Hooks 闭包陷阱资料" })
        └── ResearchAgent (独立 streamText)
              ├── systemPrompt: 只读研究 Agent
              ├── tools: [assessMastery]
              ├── maxSteps: 5
              └── toModelOutput → 摘要（≤500 字）
```

**关键设计**：
- `delegateTask` 通过工厂函数闭包捕获 `SubagentRegistry`，无需修改上下文
- 子 Agent 使用 `deepseek-v4-flash`（比主模型便宜），降低委派成本
- `toModelOutput()` 控制返回给主 Agent 的内容，过滤详细对话过程

## 5.4 A2UI 框架（迭代 020）

AI-to-UI（A2UI）是消息渲染的抽象层，将 Agent 输出映射为结构化 UIBlock，前端通过 Registry 自动选择对应组件渲染。

### UIBlock 类型

| type | 组件 | 说明 |
|------|------|------|
| `text` | TextBlock | Markdown 文本（含代码块、行内代码） |
| `assessment` | AssessmentBlock | 学习评估卡片（包装现有 AssessmentCard） |
| `quiz` | QuizPanel | 交互式测验（可点击选项 + 即时反馈） |
| `code-result` | CodeResultBlock | 代码运行结果（代码 + stdout/stderr + 退出码） |
| `formula` | FormulaDisplay | LaTeX 公式（KaTeX 渲染） |
| `diagram` | DiagramRenderer | 图表（预留占位） |
| `table` | TableBlock | 结构化表格（迭代 024） |
| `callout` | CalloutBlock | 提示卡 tip/warning/key（迭代 024） |
| `comparison` | ComparisonCard | 对比卡 A vs B（迭代 024） |
| `heading` | HeadingBlock | 标题 h2/h3（迭代 029） |
| `badge` | BadgeBlock | 徽章标签 success/warning/info（迭代 029） |
| `mastery-report` | MasteryReportBlock | 掌握总结报告（迭代 029） |

### 渲染流程

```
Message → 检查 metadata.uiBlocks
  ├── 有 uiBlocks → 遍历 blocks，Registry 查找组件渲染
  └── 无 uiBlocks → 降级为单个 TextBlock（向后兼容）
```

### 依赖

| 包 | 用途 |
|----|------|
| `monaco-editor` + `@monaco-editor/react` | Monaco Editor 代码编辑器（Catppuccin 暖色主题） |
| `katex` | LaTeX 公式渲染 |
