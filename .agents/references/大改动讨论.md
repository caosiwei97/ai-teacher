# 大改动讨论机制

> 按需加载：当你要做大范围改动（3+ 文件）时，读取本文件。

---

## 什么是"大改动"

满足以下**任一条件**即为大改动，必须先讨论：

| 条件 | 示例 |
|------|------|
| 涉及 3 个以上文件 | 新功能通常跨多个文件 |
| 涉及 Prisma Schema 变更 | 新增/修改 model 或字段（影响迁移） |
| 涉及新增 npm 依赖 | 引入新的库或框架 |
| 涉及 API 接口变更 | 新增/修改/删除 API 路由 |
| 涉及页面路由结构变更 | 新增页面、修改 URL 结构、改变重定向逻辑 |
| 涉及 Agent 工具或 Prompt 变更 | 影响 AI 行为核心逻辑 |
| 涉及架构层面重构 | 如抽象基类、服务层拆分、跨模块影响 |

## 讨论流程

```
识别为大改动
  ↓
1. 暂停实现，向用户展示方案摘要：
   - 📋 变更范围（涉及哪些文件/模块）
   - 🏗️ 实现方案（怎么改，分几步）
   - ⚠️ 风险和副作用（可能影响什么）
   - 📝 需要同步更新的文档
  ↓
2. 等待用户确认
  ↓
3. 用户确认后 → 创建 todo list → 开始实现
```

## 小改动直接执行

以下情况可以不经讨论直接执行：

- 单文件 bug 修复（不涉及其他文件）
- UI 文案调整
- CSS 样式微调
- 注释修改
- 配置值微调（端口号、超时时间等）
