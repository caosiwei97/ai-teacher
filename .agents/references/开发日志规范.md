# 开发日志规范

> 按需加载：当你要写开发日志时，读取本文件。

---

## 条目格式（精简版）

```markdown
#### {标题}

**状态**：✅ 已完成 | 🔧 进行中 | ⏸️ 暂停 | ❌ 失败

**事件**：1-2 句概述（不列文件清单，`git show` / `git log --stat` 可查）

**决策**：仅记录 git 无法告诉你的 WHY（没有则省略此行）

**下一步**：{明确的下一步，如与下条连续可省略}
```

## 记录时机

- 每个独立任务完成时（不是整个迭代结束时）
- 发现 bug 时记录问题和修复方案
- 用户做出关键决策时
- session 结束前必须记录当前状态

## 日归档保留策略

- **里程碑摘要**（主文件 `开发日志.md` 的 `## 里程碑时间线` 区域）：**永久保留**，每条 1 行
- **日归档**（`开发日志/YYYY/MM/DD.md`）：**只保留最近 3 天**，超期文件在 session 启动时自动清理
- 清理时同步移除主文件归档目录表中对应条目（但里程碑摘要不动）

## 按日期归档结构

```
docs/开发/开发日志.md                ← 索引文件（里程碑摘要 + 归档目录表 + 继续点）
docs/开发/开发日志/2026/05/DD.md     ← 当日详细日志（仅保留最近 7 天）
```

## 写入规则

- 详细日志写入 `开发日志/YYYY/MM/DD.md`（当天文件）
- 主文件 `开发日志.md` 只保留：里程碑级摘要 + 归档目录表 + "下次继续点"
- 每个新 session 第一条日志写入前，检查当天日期文件是否存在，不存在则创建
- **归档目录同步**：session 结束时必须确认归档目录表中当天条目已存在且链接指向日期文件（不能为 `—`）
- **3 天清理**：session 启动时检查日归档目录，删除超过 3 天的文件，并同步移除主文件归档目录表中对应条目（里程碑摘要不动）
- 归档目录表格式：

```markdown
| 日期 | 文件 | 主要内容 |
|------|------|----------|
| 2026-05-08 | [08.md](开发日志/2026/05/08.md) | 项目初始化 ~ 迭代 010 |
```
