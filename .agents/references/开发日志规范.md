# 开发日志规范

> 按需加载：当你要更新开发日志时，读取本文件。

---

## 用途

`docs/开发/开发日志.md` 仅用于 **session 断点恢复**，不记录开发历史（那是 git 的事）。

## 文件结构

```markdown
## 待同步文档变更
<!-- 代码改了但文档未同步时追加条目，下个 session 消费 -->

## 下次 Session 应从这里继续
- **当前进度**：{最后完成的迭代/任务}
- **下一步**：{明确的下一步动作}
- **注意**：{阻塞项或需要留意的事项，没有则省略}
```

## 写入规则

- "下次继续"区域保持 2-5 行，只写**动作指引**，不写项目基础信息（技术栈、端口等属于 AGENTS.md）
- 每次 session 结束时覆盖更新（不是追加）
- "待同步文档变更"由 Agent 按需追加，下个 session 消费后清空
