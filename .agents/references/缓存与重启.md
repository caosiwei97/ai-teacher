# 缓存清除与服务重启

> 按需加载：当你涉及 CSS/env/config/prisma 变更时，读取本文件。

---

## 必须清除 `.next/` 缓存的场景

```bash
rm -rf apps/web/.next
pnpm dev   # 重启 dev server
```

| 触发条件 | 为什么需要清除 |
|---------|-------------|
| 修改 `globals.css` 的 `@theme` 变量或 `@layer` 定义 | Tailwind CSS 4 缓存旧的 CSS 变量值 |
| 修改 `postcss.config.mjs` 或 `tailwind.config.*` | PostCSS 配置变更不触发 HMR |
| 修改 `.env` 或环境变量 | Next.js 在启动时读取环境变量，运行时不会重新读取 |
| 修改 `prisma/schema.prisma` 并执行 migrate | Prisma Client 缓存需要重新生成 |
| 修改 `next.config.*` | Next.js 配置变更需要重启 |
| 页面路由或布局结构发生结构性变化 | Next.js App Router 缓存了路由映射 |

## 必须重启 dev server 的场景

| 触发条件 | 说明 |
|---------|------|
| 新增/删除 npm 依赖 | 需要重启让新依赖生效 |
| 修改 `tsconfig.json` | TypeScript 编译选项变更 |
| Prisma migrate 后 | 生成的 Client 需要新进程加载 |
| 添加新的 API route 文件（首次） | 部分情况下 Next.js 不会自动检测新文件 |

## 代码变更后的验证流程

```
代码变更
  ↓
1. 判断是否触发缓存清除（查上表）
  ↓
2a. 不触发 → 依赖 HMR 热更新，直接验证
2b. 触发 → rm -rf apps/web/.next → 重启 dev server
  ↓
3. 构建验证：pnpm --filter @ai-teacher/web build
  ↓
4. E2E 验证（发版时）：npx playwright test
```
