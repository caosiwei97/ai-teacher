# 质量门控

> 按需加载：当你要提交代码或跑测试时，读取本文件。

---

## 构建检查（每次 commit 前必须通过）

```bash
pnpm --filter @ai-teacher/web build
```

## E2E 测试（发版时运行，日常不强制）

```bash
pnpm test:e2e
```

> E2E 测试需要启动完整的 Web + Server + Worker + 数据库环境，日常频繁提交时跑 E2E 会严重影响开发效率。

## E2E 测试同步规则

当需求变更涉及 UI 行为、页面路由、API 接口变化时，**必须同步修改对应的 E2E 测试用例**。

先查迭代 ↔ E2E 映射表（在 `docs/开发/迭代计划.md` 的"迭代 ↔ E2E 测试映射"章节）。

| 变更类型 | 必须检查/修改的 E2E 文件 |
|---------|----------------------|
| 修改页面路由或重定向行为 | `e2e/home.spec.ts` 及涉及路由的测试 |
| 修改 Chat UI 交互 | `e2e/chat.spec.ts`, `e2e/learn.spec.ts` |
| 修改诊断流程 | `e2e/diagnostic.spec.ts` |
| 修改会话管理（新建/归档/状态） | `e2e/session-management.spec.ts` |
| 修改掌握度/节点推进逻辑 | `e2e/mastery.spec.ts` |
| 修改快问/建议回复功能 | `e2e/quick-features.spec.ts` |
| 新增 API 路由 | 新增对应 `e2e/*.spec.ts` 或在已有文件中追加用例 |
| 修改推荐话题/首页内容 | `e2e/home.spec.ts` |

## E2E 测试分类速查

| 分类 | 名称 | 测试文件 |
|------|------|---------|
| A | 新会话与诊断 | `e2e/home.spec.ts`, `e2e/diagnostic.spec.ts` |
| B | 路线图与学习流程 | `e2e/learn.spec.ts` |
| C | 苏格拉底式教学循环 | `e2e/chat.spec.ts`, `e2e/learn.spec.ts` |
| D | 评估与门控 | `e2e/mastery.spec.ts` |
| E | 会话管理 | `e2e/session-management.spec.ts` |
| F | 输入区功能 | `e2e/quick-features.spec.ts` |
| G | 右侧栏交互 | `e2e/learn.spec.ts` |
| I | 布局与导航 | `e2e/learn.spec.ts` |
| J | 流式响应与性能 | `e2e/chat.spec.ts` |
| K | 设置页 | `e2e/settings.spec.ts` |

## E2E 失败处理

1. 分析失败原因：代码 bug 还是测试用例过时？
2. 代码 bug → 修复代码 → 重新跑 E2E → 通过后提交
3. 测试过时 → 确认新行为符合预期 → 更新测试 → 重新跑 E2E → 通过后提交
4. 绝不允许：跳过测试、删除失败测试、@ts-ignore 测试错误
